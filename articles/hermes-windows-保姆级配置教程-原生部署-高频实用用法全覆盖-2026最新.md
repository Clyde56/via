## 五、进阶优化：迁移目录\+提速设置

默认安装在 C 盘，长期使用会堆积日志和缓存文件，建议手动迁移至其他磁盘，避免 C 盘爆满。

1. 新建非 C 盘目录，例如 `D:\\hermes\\data`、`D:\\hermes\\logs`

2. 执行 `hermes config open` 打开配置文件

3. 修改 `data\_path`、`log\_path` 为新目录路径

4. 执行 `hermes restart` 重启服务生效

优化后可大幅减少 C 盘占用，同时提升日志读写、任务运行速度。

## 六、高频踩坑问题与解决方案（2026实测）

### 1\. 提示“hermes 不是内部或外部命令”

**原因**：环境变量未刷新、安装未完成

**解决**：关闭所有终端，重新打开 PowerShell；若无效，重新执行安装脚本覆盖安装。

### 2\. 安装脚本执行失败、网络超时

**原因**：GitHub 源访问不稳定

**解决**：切换国内镜像源安装，或临时开启网络代理，重新执行安装命令。

### 3\. hermes doctor 检测异常、服务启动失败

**原因**：配置文件损坏、权限不足

**解决**：执行 `hermes config reset` 重置默认配置，重启终端后重新初始化。

### 4\. 日志乱码、中文显示异常

**解决**：PowerShell 执行编码设置，固定 UTF\-8 编码，重启服务即可。
